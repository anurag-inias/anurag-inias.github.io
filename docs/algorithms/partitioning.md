    ```java linenums="1"
    int partition(int[] array) {
      int pivot = array[0];
      int cursor = 1;
      
      for (int i = 1; i < array.length; i++)
        if (array[i] < pivot)
          swap(array, cursor++, i);

      swap(array, 0, cursor-1);
      return cursor-1;
    }
    ```

So what's happening here?

- We pick the first element (`idx = 0`) of the array as pivot.
- Index `0` is omitted from the process.
- The `cursor` is where the next element `< pivot` will be placed.

When the whole loop is done, we are left with `cursor` pointing to where the next `< pivot` element would've gone. At this stage we have:

- `idx = 0` where pivot is,
- `idx = [1, cursor)` where elements `< pivot` are,
- and `idx = [cursor, array.length)` where elements `≥ pivot` are.

So a final swap is needed b/w `0` and `cursor-1`. This splits the array in two parts:

- $\big[0, \ cursor\big)$ all $a_i < pivot$.
- $\Big[cursor, \ len(array)\Big)$ all $a_i \ge pivot$.


## Partition in three chunks

This extends the idea of previous algorithm, and uses two cursors (`l` and `r`) instead of just one.

=== "Implementation"

    ```java linenums="1"
    // Half open interval [l, c) where elements equals to pivot are grouped.
    public static int[] partition3way(int[] array) {
      int pivot = array[0];
      int l = 0, r = array.length-1, c = 0;

      while (c <= r) {
        switch (Integer.compare(array[c], pivot)) {
          case -1 -> swap(array, l++, c++);
          case 1 -> swap(array, c, r--);
          default -> c++;
        }
      }

      return new int[]{l, c};
    }
    ```

Note interesting invariants here: 

- case $a_c < pivot$ and case $\text{default}$ ensures that $c \ge l$ at all times.
- `l` always point to an element which is in the middle slice, i.e. `== pivot`.
- `c` is always pointing to the next element which needs to be processed, and in the ends point to the first element of last slice, i.e. `> pivot`.
