# Find the next permutation

## Problem

Given an array of numbers, find the next permutation that would follow it in sorted order.

## Examples

Input | Output
------|-------
[1, 2, 3] | [1, 3, 2]
[2, 3, 1] | [3, 1, 2]
[3, 2, 1] | [1, 2, 3] 
[1, 1, 5] | [1, 5, 1]
[4, 2, 0, 2, 3, 2, 0] | [4, 2, 0, 3, 0, 2, 2]

!!! note
    We are not looking for the closest neighbour in the permutation space. It should be the permutation with least difference.

??? tip "Solution"

    The idea is to "dial up" followed by a "dial down".

    1. We first find the lowest possible number which can be swapped by a number larger than it to it's right. For example: $[...4...9] \implies [...9...4]$.

    2. The resulting number has jumped ahead a bit too much. It should be "dialed back". So we need to go go down by sorting the bolded subarray $[...9\textbf{...4}]$.

    public boolean nextPermutation(int[] nums) {
      if (nums == null || nums.length < 2) {
        return false;
      }

      for (int i = nums.length - 2; i >= 0; i--) {
        // Find the smallest number to the right, which is still greater than nums[i].
        int candidate = findNext(nums, i + 1, nums.length, nums[i]);
        if (candidate >= 0) {
          swap(nums, i, candidate);
          Arrays.sort(nums, i + 1, nums.length);
          return true;
        }
      }

      Arrays.sort(nums);
      return false;
    }

    private int findNext(int[] nums, int start, int end, int threshold) {
      int min = start;
      for (int i = start; i < end; i++) {
        if (nums[i] > threshold && nums[i] < nums[min]) {
          min = i;
        }
      }
      return nums[min] > threshold ? min : -1;
    }

    private void swap(int[] nums, int i, int j) {
      int t = nums[i];
      nums[i] = nums[j];
      nums[j] = t;
    }
