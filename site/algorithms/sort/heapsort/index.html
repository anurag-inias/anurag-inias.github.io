Heapsort

  1. First turn the input array in a Max Heap. The root of the heap is index \(0\) with the largest element.

  2. Swap the indices \(0\) and \(size-1\), moving the largest element at the back.

  3. Shrink the heap from the end, thus moving the sorted element out of the heap.

  4. Call sink on the heap's new root.

  5. Back to #2.

type IntSlice []int

// Sink for MaxHeap
func (slice IntSlice) Sink(index, length int) {
  for index < length {
    max := index
    left := 2*index + 1
    right := 2*index + 2

    if left < length && slice[left] > slice[max] {
      max = left
    }
    if right < length && slice[right] > slice[max] {
      max = right
    }
    if max == index {
      return
    }
    slice[index], slice[max] = slice[max], slice[index]
    index = max
  }
}

// Heapify for max heap
func (slice IntSlice) Heapify() {
  for i := len(slice) / 2; i >= 0; i-- {
    slice.Sink(i, len(slice))
  }
}

func HeapSort(a []int) {
  if len(a) == 0 {
    return
  }
  slice := IntSlice(a)
  slice.Heapify()
  l := len(slice)

  for l > 1 {
    l--
    a[0], a[l] = a[l], a[0]
    slice.Sink(0, l)
  }
}

func TestFuzzy(t *testing.T) {
  var treat, ctrl []int
  rand := rand2.New(rand2.NewSource(100))

  for i := 0; i < 100; i++ {
    value := rand.Intn(50)
    treat = append(treat, value)
    ctrl = append(ctrl, value)
  }

  slices.Sort(ctrl)
  HeapSort(treat)

  result := fmt.Sprintf("%v", treat)
  expected := fmt.Sprintf("%v", ctrl)

  if result != expected {
    t.Fatalf("expected %s, but got %s", expected, result)
  }
}

Complexity

Heapify take \(O(n)\) time, followed by the first sink taking \(O(\log n)\) time, second sink taking \(O(\log (n-1))\) time, third sink taking \(O(\log (n-2))\) time and so on.

\[ \begin{alignat}{1} \log n + \log (n-1) + \log (n-2) + \dots + \log 1 & < \log n + \log n + \dots + \log n \\ \\ \log(n!) & < n\log n \end{alignat} \]

And so the complexity of heap sort is \(O(n + \log(n!))\), which is \(O(n\log n)\).

Scenario Comparisons
Worst \(O(n\log n)\)
Average \(O(n\log n)\)
Best \(O(n\log n)\)

Properties

  • Heap sort has a simple non-recursive impelmentation.
  • Doesn't require any auxiliary space.
  • Heapsort will reliably complete in \(O(n\log n)\) time even in the worst case, unlike Quicksort which can take \(O(n^2)\).
  • Has poor locality of reference, and as such, a well-implemented Quick sort can be \(2-3x\) faster.
